# Engenharia_de_prompt
### Prompt Mestre Inicial
"Escreva um programa em código python que funcione como uma calculadora"
### Variação 1
"Adicione a função de raiz quadrada na mesma calculadora"
**Resposta do LLM: Ótima ideia! Para adicionar a raiz quadrada, vamos precisar importar o módulo math, que já vem por padrão no Python. Ele contém a função sqrt() (abreviação de square root).
### Variação 2
"Como transformar essa calculadora em uma interface visual com botões?"
**Resposta do LLM: Para transformar sua calculadora em algo com janelas e botões, a biblioteca mais comum e nativa do Python é a Tkinter. Ela permite criar interfaces gráficas (GUIs) sem precisar instalar nada extra.
### Variação 3
"Explique quais as funções dessa calculadora"
**Resposta do LLM: Essa calculadora visual que montamos não é apenas um amontoado de botões; ela segue uma lógica de "Event-Driven Programming" (Programação Orientada a Eventos). Ou seja, o código fica parado esperando você clicar em algo para reagir.
### Reflexão
perguntas simples só entregam o básico, como exemplo do primeiro pedido onde a A.I apenas entregou
uma calculadora comum com um modelo que só conseguia resolver as quatro operações básicas sendo
elas adição, subtração, multiplicação e divisão. Porém após realizar pedidos mais específicos e
sugeridos pela própria A.I, podemos acrescentar e melhorar o mesmo programa.
A parte mais útil foi o prompt mestre, pois ele entregou um programa simples e com fácil entendimento
para iniciantes em programação, como uma base onde poderia usar ela e seguir modificando e
melhorando cada vez mais o programa com base nos meus desejos.
Essa técnica poderia ser aplicada para a criação de outro programa, onde tudo começaria da base e aos
poucos, com a ajuda da A.I. poderia complementar a base.
